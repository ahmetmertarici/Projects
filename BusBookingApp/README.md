Bus ticketing system Web application.

• Project Description: There are 2 main roles in the system (Admin and Customer). Anyone using the system can search after selecting the trip details (departure - arrival city, travel date). You can switch to seat selection. The user can get the ticket by passing the payment stage. Users who are members of the system can send their complaints about travels or other issues to the admin. Complaints appear on the user page along with the answers given by the admin. Admin can add new user, new role and change any user's role. Admins can add, edit and delete new cities. In addition, the Admin can create new travel routes, edit existing routes or delete them. For the payment process, iyzico integration was carried out into the system.

• Technologies Used: C# .Net, ASP Net Core(MVC), ASP Net Core Identity, EF Core CodeFirst, Bootstrap, Javascript, CSS3, HTML5, SQLite, Generic Repository Pattern, Layered Architecture, OOP,
